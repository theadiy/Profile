Web Application Security
1. IDOR – Insecure Direct Object Reference
The application exposes internal object references, like user IDs, without proper authorization checks.
URL: /profile/123
Change to /profile/124 → Accessing someone else’s profile
2. Stored XSS – Comment Section Injection
Attacker injects a JavaScript payload into a persistent field. When other users visit the page, the script executes in their browser.
Input: <script>alert('XSS')</script>
Rendered unsanitized in blog comments section.
